# `.colmesh` File Format

Version 1. Big-endian binary (N64 native, matches `.lvl` convention).

Generated by `tools/colmesh_bake.py`. Consumed at runtime by
`src/user/gameplay/physics/coll_mesh.cpp` (Inc 3).

## Header (72 bytes, at offset 0)

| Offset | Type         | Field              | Notes |
|--------|--------------|--------------------|-------|
| 0      | char[4]      | magic              | `"CMSH"` |
| 4      | uint16       | version            | `1` |
| 6      | uint16       | flags              | bit0 = has_bvh |
| 8      | int16[3]     | aabb_min           | mesh AABB min, quantized |
| 14     | int16[3]     | aabb_max           | mesh AABB max, quantized |
| 20     | float        | quant_scale        | see §Quantization |
| 24     | float[3]     | quant_origin       | see §Quantization |
| 36     | uint32       | vertex_count       | ≤ 65535 |
| 40     | uint32       | triangle_count     | ≤ 32767 (so 2N-1 BVH nodes ≤ 65535) |
| 44     | uint32       | bvh_node_count     | ≤ 65535 |
| 48     | uint32       | surface_link_count |  |
| 52     | uint32       | vertex_offset      | byte offset from file start; 8-byte aligned |
| 56     | uint32       | triangle_offset    | 8-byte aligned |
| 60     | uint32       | bvh_offset         | 8-byte aligned |
| 64     | uint32       | surface_link_offset| 8-byte aligned |
| 68     | uint8[4]     | pad                | zeroed |

## Vertices (6 bytes each)

```c
struct CollVertex { int16_t pos[3]; };
```

Stored at `vertex_offset`.

## Triangles (12 bytes each)

```c
struct CollTriangle {
    uint16_t i0, i1, i2;    // vertex indices
    uint16_t material;       // bit flags (see §Material)
    uint16_t face_id;        // == array index (bake tool asserts this)
    uint16_t pad;            // MUST be 0
};
```

Stored at `triangle_offset`. `face_id == array index` is a hard invariant.

## BVH nodes (16 bytes each)

```c
struct CollBvhNode {
    int16_t  aabb_min[3];    // quantized
    int16_t  aabb_max[3];    // quantized
    uint16_t left_or_first;  // see §BVH Layout
    uint16_t count_or_zero;  // 0 = internal; > 0 = leaf triangle count
};
```

Stored at `bvh_offset`. Depth-first, left subtree before right.

## Surface links (4 bytes each, sorted by face_id)

```c
struct CollSurfaceLink {
    uint16_t face_id;
    uint16_t owner_id;       // 0xFFFF (INVALID_OWNER) if static
};
```

Stored at `surface_link_offset`. Sparse — only moving-platform faces.

---

## Quantization

World position is reconstructed as:

```
world_pos = quant_origin + int16_pos * quant_scale
```

All three fields (`aabb_min`, `aabb_max`, and all `CollVertex.pos` and
`CollBvhNode.aabb_*`) are in the same quantized space.

`quant_scale` is chosen so that all source positions fit in `[-32767, 32767]`
after `(pos - quant_origin) / quant_scale`.

---

## BVH Layout (skip-pointer scheme)

Nodes are stored depth-first. For an internal node at index `idx`:

```
left_child  = idx + 1
right_child = idx + node.left_or_first   // skip-pointer past left subtree
```

For a leaf node (`count_or_zero > 0`):

```
triangles[left_or_first .. left_or_first + count_or_zero - 1]
```

Traversal pseudocode (stack `uint16_t stack[32]`):

```c
stack[0] = 0; top = 1;
while (top > 0) {
    uint16_t idx = stack[--top];
    if (!ray_vs_aabb(nodes[idx])) continue;
    if (nodes[idx].count_or_zero > 0) {  // leaf
        for (int t = nodes[idx].left_or_first;
             t < nodes[idx].left_or_first + nodes[idx].count_or_zero; t++)
            test_triangle(t);
    } else {  // internal
        stack[top++] = idx + nodes[idx].left_or_first;  // right child
        stack[top++] = idx + 1;                          // left child
    }
}
```

Stack depth ≤ 32 is guaranteed because the bake tool enforces depth ≤ 30.

## Material Flags

| Bit  | Value  | Name       | Meaning |
|------|--------|------------|---------|
| 0    | 0x0001 | solid      | blocks movement |
| 1    | 0x0002 | oneway     | resolves only when sweep direction · normal < 0 |
| 2    | 0x0004 | death      | kills player on contact; motor reports, game reacts |
| 3    | 0x0008 | climbable  | sets wall_normal + face_id for grab logic |
| 4    | 0x0010 | ice        | friction = 0 in motor tangent damping |

Applied from `.map` texture name suffix at bake time:
`_solid`, `_oneway`, `_death`/`_kill`, `_climbable`, `_ice`.
Default (no suffix) = `solid`.

---

## Size Budget

Per-level target: ≤ 256 KB. `tools/colmesh_bake.py` prints a warning and
describes the NO-GO fallback (see `data-model.md §10`) if exceeded.
