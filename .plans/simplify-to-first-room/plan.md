# Simplify to first-room level

## Context

The OG 1-1 map conversion is complex (102 faces, fragile winding fix, unused brush
classes). The user wants a stable, project-owned level that keeps the textured
render pipeline, CollMesh collision, and physics motor — without the OG map
complexity.

The hand-crafted `assets/rooms/first-room/first-room.map` already exists and bakes
correctly (54 faces, 2 entities). The ROM just hardcodes a load of `1-1.lvl`.
This plan switches to first-room as the default level and removes the 1-1
dependency from the build.

## Architectural decisions

- Switch default level from `1-1` (OG Quake map) to `first-room` (project-owned).
- Keep the full bake pipeline: .map → .lvl → .colmesh. No pipeline changes.
- `rock_1_climbable.sprite` created as a copy of `rock_1.sprite` for the climb wall.
- Unused OG textures (`snow_1`, `rock_2`) removed from DFS to reduce ROM size.
- LevelRenderer + MaterialCatalog + CollMesh all stay as-is.

## Risks accepted

- **first-room winding**: small set of axis-aligned brushes, unlikely to have the
  winding issues that plagued 1-1. Verify visually after ROM boot.
- **Spawn position**: 12.8 is on the platform surface (platform is z=[0,32] Quake
  → Y=[0,6.4] port). With half_height+skin lift applied in Init(), the player
  will be correctly placed.
- **rock_1_climbable.sprite missing from DFS**: MaterialCatalog skips it
  gracefully, but climb wall faces will have material_id=1 with no texture.
  Mitigation: create the sprite copy.
